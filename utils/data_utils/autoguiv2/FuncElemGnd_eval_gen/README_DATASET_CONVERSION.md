# Converting FuncElemGnd Questions to Hugging Face Dataset

This guide explains how to use `3_convert_to_hf_dataset.py` to convert the generated questions into a Hugging Face dataset format and upload it.

## Prerequisites

Install required packages:
```bash
pip install datasets huggingface_hub pillow
```

## Dataset Structure

The script converts the question generation output into a structured dataset where each entry contains:

- **image**: The actual image (if `--include-images` is set) or path
- **image_path**: Full path to the image file
- **image_key**: Relative key used in the JSON files
- **question**: Natural language question about element functionality
- **action_intent**: Precise action description for locating the element
- **action_type**: Type of interaction (clicking, hovering, typing, etc.)
- **group_index**: Index of the similar element group
- **visual_similarity**: Description of visual similarity between elements
- **target_element_id**: ID of the target element within the group
- **target_element_bbox**: Bounding box [x_min, y_min, x_max, y_max] (normalized 0-1000)
- **target_element_description**: Detailed description of the target element
- **target_element_functionality**: Unique functionality of the target element
- **target_element_interaction_outcomes**: JSON string of interaction outcomes
- **similar_elements**: JSON string containing info about all similar elements in the group
- **num_similar_elements**: Number of visually similar elements in the group
- **omniparser_type**: Element type from OmniParser (e.g., 'text', 'icon', 'button')
- **omniparser_content**: Text content extracted by OmniParser
- **omniparser_interactivity**: Whether OmniParser detected the element as interactive
- **omniparser_source**: Source of the OmniParser detection
- **omniparser_bbox**: OmniParser bounding box [x_min, y_min, x_max, y_max] (normalized 0-1)

## Usage

The script always saves the dataset locally as cache first, then uploads to Hugging Face Hub. This ensures you have a local backup even if the upload fails.

### Basic Usage

Convert and upload to Hugging Face Hub:

```bash
python 3_convert_to_hf_dataset.py \
    --questions-file /path/to/FuncElemGnd/grounding_questions.json \
    --repo-id username/funcelem-grounding \
    --hf-token YOUR_HF_TOKEN \
    --include-images
```

### Private Dataset

To make the dataset private:

```bash
python 3_convert_to_hf_dataset.py \
    --questions-file /path/to/FuncElemGnd/grounding_questions.json \
    --repo-id username/funcelem-grounding \
    --hf-token YOUR_HF_TOKEN \
    --private \
    --include-images
```

## Local Cache

The dataset is always saved locally to a `hf_dataset_cache` directory in the same location as your questions file. This serves as a backup and allows you to retry uploads if they fail.

**Cache location**: `[questions_file_directory]/hf_dataset_cache/`

## Command-Line Arguments

### Required Arguments

- `--questions-file`: Path to the questions JSON file generated by `2_generate_func_elemgnd_questions.py`
- `--repo-id`: Hugging Face repository ID (format: `username/dataset-name`)

### Optional Arguments

- `--hf-token`: Hugging Face authentication token (optional if you've run `huggingface-cli login`)
- `--private`: Make the dataset private on Hugging Face Hub

### Processing Arguments

- `--include-images`: Include actual image data in the dataset (recommended for HF upload, increases dataset size)

## Authentication

### Hugging Face Token

You can provide your HF token in three ways:

1. **Via command line**: `--hf-token YOUR_TOKEN`
2. **Via environment variable**: Set `HF_TOKEN=YOUR_TOKEN`
3. **Via huggingface-cli**: Run `huggingface-cli login` once

To get your token:
1. Go to https://huggingface.co/settings/tokens
2. Create a new token with write access
3. Copy and use it

## Loading the Dataset

After uploading to Hugging Face, you can load the dataset as follows:

```python
from datasets import load_dataset

# Load from Hub
dataset = load_dataset("username/funcelem-grounding")

# Access a sample
sample = dataset['train'][0]
print(f"Question: {sample['question']}")
print(f"Action: {sample['action_type']}")
print(f"Target bbox: {sample['target_element_bbox']}")
print(f"OmniParser type: {sample['omniparser_type']}")
print(f"OmniParser content: {sample['omniparser_content']}")

# Display the image
sample['image'].show()

# Parse JSON fields
import json
interaction_outcomes = json.loads(sample['target_element_interaction_outcomes'])
similar_elements = json.loads(sample['similar_elements'])
```

Or load from local directory:

```python
from datasets import load_from_disk

dataset = load_from_disk("/path/to/output/hf_dataset")
```

## Notes

### Image Inclusion

- **With `--include-images`**: The dataset will contain the actual image data. This makes the dataset larger but more portable and easier to use.
- **Without `--include-images`**: Only image paths are stored. The dataset is smaller, but users need access to the original images.

For Hugging Face Hub uploads, it's recommended to use `--include-images` for better usability.

### Bounding Box Formats

The dataset includes two bbox formats:
1. **target_element_bbox**: Normalized to 0-1000 scale (from the question generation)
2. **omniparser_bbox**: Normalized to 0-1 scale (from OmniParser)

### OmniParser Matching

The script automatically matches target elements with OmniParser detections using IoU (Intersection over Union). If no good match is found (IoU < 0.3), the OmniParser fields will be empty strings or default values.

## Troubleshooting

### "Image not found" errors
- Ensure `--image-src-dir` matches the directory structure from question generation
- Check that image paths in the questions file are correct

### Authentication errors
- Verify your HF token has write permissions
- Try running `huggingface-cli login` first

### Out of memory errors
- Process a subset of the data first
- Consider not including images (`--include-images` flag omitted) for very large datasets
- Use a machine with more RAM

### OmniParser data not found
- The script looks for OmniParser results in `[base_dir]/omniparser/[image_stem].json`
- Ensure OmniParser results are available in the expected location
- Missing OmniParser data is not fatal; those fields will be empty
